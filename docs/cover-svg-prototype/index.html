Honest notes

What works. The corpus reads as a coherent set — shared rust palette + topic-hue secondary keeps them "siblings". Compositions differ enough per topic that they don't blur together on the index. All six loaded instantly (<3 kB each). In V1 cards the aesthetic holds up.

What's risky. Claude's SVG drafting is hit-or-miss without strong style guardrails. I wrote each one deliberately; a cold run on a new topic might produce something off-brand or cluttered. Style consistency depends on the SKILL prompt being tight.

Topic failure modes. Highly abstract topics ("philosophy of X", "survey of Y") are hardest — no concrete motif to lean on. For those, fall through to V6 typographic fallback.

Next step if keeping. Extend skills/scout/SKILL.md with a Cover image section: style guide (palette, no-text rule, composition inspiration per topic kind), output contract (write cover.svg, add cover: cover.svg to frontmatter), fallback instruction (skip if unsure). Then update the Atlas card partial to use {{ page.cover }} with the tag-keyed curated set + V6 typographic as cascade.
