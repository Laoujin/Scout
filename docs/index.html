Two repos, one flow

Scout researches. Atlas publishes.

You own both. One does the work, the other shows the result.

S
The researcher

Scout

A Docker runner on your NAS, a GitHub Action, and a Claude-Code-based skill that knows how to cite. Give it a topic, depth, and format.

  • 3 depths: ceo, standard, deep
  • 2 formats: .md or .html
  • WebSearch + WebFetch + Playwright fallback
  • 5–30 min per topic
git push
A
The shelf

Atlas

A Jekyll site, served by GitHub Pages. Every research result lands in research/DATE-SLUG/ with frontmatter; the index auto-builds.

  • Cards with cover art and tag filtering
  • Inline citations on every claim
  • Zero ops — GitHub rebuilds on push
  • Yours to fork and customize
Why Scout

A research tool that doesn't lie.

Four principles that fell out of using LLMs for real research work.

01

Cited per-line

Every factual claim carries its source URL inline. No orphan "references" dump at the bottom. If a claim has no URL, it doesn't get written.

02

Your hardware

Runs in Docker on any always-on Linux device with AVX2 — an old NAS, a mini-PC, a Raspberry Pi 5. No managed service, no vendor lock-in.

03

Three depths

ceo for a one-page decision, standard for 2–4 pages with trade-offs, deep for every angle. You pick per-topic.

04

Yours to fork

Every part of the pipeline is in two repos you own. Change the skill, the layout, the cards, the palette. Nothing phones home.

How it works

Six boxes, one arrow chain.

You open a GitHub issue. The Action fires on your NAS. Scout researches. Atlas gets a push. Pages rebuilds. Done.

01 → 02

Open an issue

Title is your topic. Label sets the depth — ceo, standard, or deep. Works from mobile; one-handed research queueing on the train.

03

Scout does the work

Runs on your hardware. Hits WebSearch, WebFetch, Playwright as needed. Writes the artifact with inline citations — no claim without a URL.

04 → 06

Atlas publishes

Scout pushes to research/DATE-SLUG/index.{md,html}. GitHub Pages rebuilds. You read it at your own domain.

Install

Three steps. Twenty minutes.

Assumes you have a Linux device that can run Docker, and a GitHub account.

1

Fork the templates

Scout (the runner) and Atlas (the site) are two repos. Fork both with one click.

Fork Scout ↗ Fork Atlas ↗
2

Pick your look

Palette, page layout, card style. The picker below gives you a config string to paste into the install script.

Open the picker ↓
3

Run install.sh

SSH to your always-on device. Paste the command — walks you through the runner token and deploy key interactively.

See the command ↓
$ curl -fsSL https://raw.githubusercontent.com/Laoujin/Scout/main/install.sh \
| bash -s -- --config=s1.rust.v1
# prompts you for the runner token + deploy key interactively.
# pick your config above — the flag pre-seeds Atlas with that look.
